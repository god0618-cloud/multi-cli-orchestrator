Project Design Brief · v5.0.0

Multi-CLI Orchestrator

本地优先的多 AI Coding CLI 协作控制平面：把多个 CLI 工位组织成可监管、可审计、可回放、可停止的小团队。

v5.0.0 ReleasedStrict-Gate LoopCI Passed

1. 项目定位

它不是替代任意 CLI 的大一统 agent，而是一个控制面：负责任务状态、adapter 能力、sandbox 边界、workflow gate、artifact evidence、run ledger、dashboard 和 replay。

2. 设计原则

本地优先

任务状态和证据优先写入本地 workspace。

证据优先

完成声明必须有 artifact / ledger / report 支撑。

可停止

缺证据 wait，失败 escalate，完成 complete。

3. v5.0 核心命令

mco workflow observe <task_id>
mco workflow loop <task_id> --max-steps 1
mco orchestrate-start "Strict product task" --template strict-self-closing
推荐动作含义
advance当前 phase gate 满足，可以推进。
wait缺证据或 dispatch 未结束。
escalate失败、阻塞或需要用户决策。
completeworkflow 已完成。

4. strict-self-closing 工作流

plan
→
execute
→
verify
→
close

每个阶段都必须满足证据门禁，不能靠 prompt 口头约定推进。
