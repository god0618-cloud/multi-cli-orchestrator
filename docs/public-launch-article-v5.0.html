01 / Why

多 CLI 好用，但协作需要控制面

Codex、Claude Code、Kimi Code、Mimo Code、CodeWhale 等工具各有优势。复杂任务真正的问题不是“能不能让 agent 干活”，而是谁在干、能不能自动干、失败后是否停止、证据在哪、什么时候必须找人拍板。

状态外置

任务状态写进 workspace，不靠聊天窗口记忆。

证据优先

产物必须登记 artifact，过程写进 RUN_LEDGER。

门禁推进

phase 是否推进由 gate 判断，而不是 prompt 口头约定。

02 / v5.0

从派棒到严格门禁自闭环

mco workflow observe <task_id>
mco workflow loop <task_id> --max-steps 1
动作含义
advance证据满足，可以推进。
wait证据不足或 worker 未结束。
escalate失败、阻塞或需要用户决策。
complete工作流闭环完成。
03 / Template

strict-self-closing 模板

plan
→
execute
→
verify
→
close

每一段都需要 durable evidence：implementation report、verification report、close report、ledger event、dashboard、dispatch clean state。缺证据就 wait，失败就 escalate。

04 / Control Room

老板视角：看得见为什么继续或停下

Workflow Loop Control

dashboard 显示当前 phase、推荐动作、推荐原因和 gate 明细。

Adapter Matrix

显式标出 execution_mode、automation_posture 和 recommended_use，Mimo / CodeWhale 目前不会被自动派发。
